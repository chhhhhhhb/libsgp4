# libsgp4

[![C++17](https://img.shields.io/badge/C%2B%2B-17-blue.svg)](https://en.cppreference.com/w/cpp/17)
[![License](https://img.shields.io/badge/License-Apache%202.0-green.svg)](LICENSE)

SGP4/SDP4 orbital propagation library for satellite tracking — a modern C++17 port.

`libsgp4` computes Earth-centered inertial (ECI) position and velocity from NORAD two-line element (TLE) sets using the SGP4 (Simplified General Perturbations 4) model. It implements the full **Spacetrack Report #3** algorithm with **WGS-84 / EGM-96** gravity model constants.

> **Authors**
>
> Original work by [Daniel Warner](https://github.com/dnwrnr/sgp4) (2013).
> Modified and maintained by [**chhhhhhhb**](https://github.com/chhhhhhhb).
>
> **Key modification:** The geodetic model has been upgraded from **WGS-72** to **WGS-84** throughout:
> - Flattening factor $1/f$: $298.26$ → $298.257223563$
> - Earth radius $R_e$ and $\mu$ already conform to WGS-84
> - $J_2$, $J_3$, $J_4$ values now use EGM-96 coefficients consistent with WGS-84

---

## Background

The SGP4 model is the standard for propagating satellite orbits from TLE data. TLEs are "mean" elements that have been generated by removing short-periodic and long-periodic perturbations. SGP4 re-applies these perturbations in a compatible way to recover the osculating position and velocity at a given time.

### Orbit Regimes

| Regime | Period | Model | Key Features |
|--------|--------|-------|-------------|
| Near-Earth | < 225 min | **SGP4** | J₂/J₃/J₄ perturbations + atmospheric drag (power-law density model) |
| Deep-Space | ≥ 225 min | **SDP4** | All SGP4 terms + lunar/solar gravity + 12h/24h geopotential resonances |
| Low Perigee | < 220 km | **Simplified SGP4** | Truncated drag terms for computational efficiency |

---

## Features

- **Full SGP4/SDP4 implementation** per Hoots & Roehrich (1980) and Vallado et al. (2006)
- **WGS-84 / EGM-96** Earth model ($R_e = 6378.137$ km, $1/f = 298.257223563$, $J_2/J_3/J_4$ from EGM-96)
- **TLE parsing** with checksum validation
- **NRLMSISE-00** (power-law) atmospheric density model for drag
- **Deep-space resonance** integrator for 12h and 24h synchronous orbits
- **Coordinate transformations**: ECI ↔ Geodetic (WGS-84 ellipsoid), GMST/LMST time conversions
- **Exception hierarchy**: `TleException`, `SatelliteException`, `DecayedException`
- **Static and shared library** builds via CMake
- **Comprehensive test suite** covering all orbit regimes
- **C++17** with no mandatory external dependencies

## Quick Start

### Prerequisites

- CMake ≥ 3.14
- C++17 compiler (GCC 8+, Clang 7+, MSVC 2019+)
- **Windows**: [MinGW-w64](https://www.mingw-w64.org/) or Visual Studio

### Build & Install

```bash
# Clone and configure
git clone https://github.com/your-org/libsgp4.git
cd libsgp4
cmake -B build -G "MinGW Makefiles"          # Windows/MinGW
# cmake -B build                              # Linux/macOS

# Build
cmake --build build

# Outputs
#   lib/libsgp4.a       — static library
#   lib/libsgp4.dll     — shared library (Windows)
#   lib/libsgp4.so      — shared library (Linux/macOS)
#   bin/sgp4_test.exe   — test executable

# Install (optional)
cmake --install build --prefix /path/to/install
```

### Basic Usage

```cpp
#include <SGP4.h>
#include <iostream>

int main()
{
    // ISS (ZARYA) TLE
    const libsgp4::Tle tle("ISS (ZARYA)",
        "1 25544U 98067A   24200.54791667  .00014344  00000+0  25520-3 0  9998",
        "2 25544  51.6413 205.8746 0002383  80.0434  20.5256 15.50138786467815");

    // Initialize the propagator
    libsgp4::SGP4 sgp4(tle);

    // Find position 60 minutes after epoch
    libsgp4::Eci eci = sgp4.FindPosition(60.0);

    libsgp4::Vector pos = eci.Position();
    libsgp4::Vector vel = eci.Velocity();

    std::cout << "Position (km):  X=" << pos.x
              << "  Y=" << pos.y
              << "  Z=" << pos.z << std::endl;
    std::cout << "Velocity (km/s): X=" << vel.x
              << "  Y=" << vel.y
              << "  Z=" << vel.z << std::endl;

    // Convert to geodetic coordinates
    libsgp4::CoordGeodetic geo = eci.ToGeodetic();
    std::cout << geo << std::endl;

    return 0;
}
```

Compile and link:

```bash
g++ -std=c++17 -I./include -L./lib myapp.cpp -lsgp4 -o myapp
```

## API Overview

### Core Classes

| Class | Header | Description |
|-------|--------|-------------|
| `Tle` | [`Tle.h`](include/Tle.h) | Parse and validate NORAD two-line element sets |
| `SGP4` | [`SGP4.h`](include/SGP4.h) | Main propagator — `FindPosition()` returns ECI state |
| `OrbitalElements` | [`OrbitalElements.h`](include/OrbitalElements.h) | J₂-corrected mean elements recovered from TLE |
| `Eci` | [`Eci.h`](include/Eci.h) | Earth-centered inertial position, velocity, and datetime |
| `CoordGeodetic` | [`CoordGeodetic.h`](include/CoordGeodetic.h) | Geodetic coordinates (lat/lon/alt) on WGS-84 ellipsoid |
| `Vector` | [`Vector.h`](include/Vector.h) | 4-component vector (x, y, z, w) with magnitude and dot product |
| `DateTime` | [`DateTime.h`](include/DateTime.h) | Julian date, GMST/LMST, epoch arithmetic |
| `TimeSpan` | [`TimeSpan.h`](include/TimeSpan.h) | Time difference representation |
| `Util` | [`Util.h`](include/Util.h) | Angle wrapping, trig utilities, radian ↔ degree conversion |

### Exception Classes

| Exception | Header | Thrown When |
|-----------|--------|-------------|
| `TleException` | [`TleException.h`](include/TleException.h) | Invalid TLE format or checksum |
| `SatelliteException` | [`SatelliteException.h`](include/SatelliteException.h) | Propagation error (eccentricity out of range, elsq ≥ 1, etc.) |
| `DecayedException` | [`DecayedException.h`](include/DecayedException.h) | Satellite perigee below Earth surface ($r_k < 1$ ER) |

### Key Methods

```cpp
// TLE parsing
libsgp4::Tle tle(name, line1, line2);
tle.Name();           tle.NoradNumber();
tle.Epoch();          tle.MeanMotion();    // rev/day
tle.Eccentricity();   tle.Inclination();   // rad
tle.BStar();          tle.ArgPerigee();    // rad

// SGP4 propagation — two interfaces
libsgp4::SGP4 sgp4(tle);
sgp4.FindPosition(tsince_minutes);                         // by time offset
sgp4.FindPosition(datetime);                               // by absolute time
sgp4.SetTle(another_tle);                                  // re-initialize

// Coordinate conversion
libsgp4::Eci eci(datetime, latitude, longitude, altitude);  // Geo→ECI
libsgp4::CoordGeodetic geo = eci.ToGeodetic();              // ECI→Geo
```

## Physical Model

### Gravity

The geopotential is modeled with zonal harmonics up to $J_4$ (EGM-96):

| Constant | Value | Description |
|----------|-------|-------------|
| $\mu$ | $398600.5$ km³/s² | Earth gravitational parameter |
| $R_e$ | $6378.137$ km | Equatorial radius (WGS-84) |
| $1/f$ | $298.257223563$ | Inverse flattening (WGS-84) |
| $J_2$ | $1.08262998905 \times 10^{-3}$ | Dynamic form factor |
| $J_3$ | $-2.53215306 \times 10^{-6}$ | Pear-shaped (north-south asymmetry) |
| $J_4$ | $-1.61098761 \times 10^{-6}$ | Higher-order flattening |

### Atmospheric Drag

Static non-rotating spherical atmosphere with power-law density:

$$\rho \propto (q_0 - s)^4, \quad q_0 = 120\text{ km},\; s_0 = 78\text{ km}$$

The $B^*$ drag term from the TLE modulates the semi-major axis decay rate.

### Deep-Space Perturbations

For periods ≥ 225 minutes, additional gravitational effects are applied:

- **Lunar/solar third-body** secular and periodic terms
- **12-hour resonance** ($2.00 < n < 2.08$ rev/day): 10-term geopotential resonance model
- **24-hour resonance** ($0.49 < n < 0.51$ rev/day): 3-term synchronous resonance model
- **Recursive integrator** with 720-minute step for accurate long-term prediction

## Running Tests

```bash
cmake -B build
cmake --build build
./bin/sgp4_test
```

The test suite covers 11 test groups:

1. **TLE Parsing** — ISS, GPS, Molniya
2. **OrbitalElements** — J₂ correction, Kepler's law consistency
3. **SGP4 Near-Space** — Forward/backward propagation, ECI↔Geodetic round-trip
4. **SDP4 Deep-Space** — 12h resonance (GPS), high-eccentricity (Molniya)
5. **DateTime Interface** — tsince vs absolute time equivalence
6. **SetTle** — Re-initialization with different satellites
7. **Error Handling** — TleException, SatelliteException, DecayedException
8. **Coordinate Transformations** — Greenwich, equator, pole benchmarks
9. **Vector Operations** — Magnitude, dot product, subtraction
10. **Constants Sanity** — Derived constant consistency (kXKE, kCK2, kCK4)
11. **TLE Checksum** — Valid TLE acceptance
